# Entrega 2 - Actualización del diseño e implementación parcial

## Propósito de la entrega

Esta entrega actualiza el documento de diseño de FinMate Mobile a partir de la realimentación del tutor. Se incorporan requerimientos codificados, diagramas UML, fichas de casos de uso, diagrama de secuencia y evidencia del avance funcional de la aplicación.

## Correcciones aplicadas

| Observación recibida | Ajuste aplicado |
|---|---|
| Agregar diagrama UML de casos de uso | Se agregó `docs/uml/casos_uso.puml`. |
| Agregar fichas de casos de uso | Se agregó `docs/fichas_casos_uso.md`. |
| Agregar diagrama de secuencia | Se agregó `docs/uml/secuencia_registrar_movimiento.puml`. |
| Usar notación UML estándar | Se documentaron actores, casos, relaciones, participantes, mensajes y clases. |
| Formalizar requerimientos | Se agregó `docs/requerimientos.md` con códigos RF y RNF. |

## Avance funcional

La aplicación mantiene una arquitectura Ionic + React + Capacitor. Para esta fase se documenta navegación principal y funcionalidad parcial en login, resumen, movimientos, registro de movimiento, metas, alertas y ajustes.

## Evidencias esperadas

1. Capturas del repositorio GitHub.
2. Capturas de los archivos agregados para Entrega 2.
3. Capturas del prototipo ejecutado en navegador.
4. Capturas de comandos de validación local.

## Nota de alcance

El APK corresponde a una fase posterior si el entorno de construcción Android se encuentra disponible. Esta entrega se concentra en actualizar el diseño técnico, evidenciar navegación funcional y preparar el proyecto para completar el avance móvil.
